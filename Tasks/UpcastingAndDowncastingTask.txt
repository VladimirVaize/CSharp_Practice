Задача: Система инвентаря для RPG

Контекст:
Мы пишем прототип RPG. У игрока есть инвентарь, в котором могут лежать разные типы предметов:
- Оружие (Weapon): Наносит урон. Имеет силу атаки (AttackPower).
- Зелье (Potion): Восстанавливает здоровье или ману. Имеет тип зелья и эффективность (Value).
- Броня (Armor): Уменьшает входящий урон. Имеет показатель защиты (Defense).

Техническое задание:
Вам дан базовый класс Item (его структуру менять нельзя, он уже утвержден архитектором):

public class Item
{
    public string Name { get; }
    public int Weight { get; }

    public Item(string name, int weight)
    {
        Name = name;
        Weight = weight;
    }

    public virtual void Use()
    {
        Console.WriteLine($"{Name} использован. Но эффекта нет (базовый класс).");
    }
}

Вам нужно:
1. Создать классы-наследники:
   - Класс Weapon, который добавляет публичное поле int AttackPower.
     Переопределить метод Use() так, чтобы он выводил: 
     "{Name} наносит {AttackPower} урона.".
   - Класс Potion, который добавляет публичное поле int HealAmount. 
     Переопределить метод Use() так, чтобы он выводил: 
     "{Name} восстанавливает {HealAmount} здоровья.".
   - Класс Armor, который добавляет публичное поле int Defense. 
     Переопределить метод Use() так, чтобы он выводил: 
     "{Name} дает {Defense} защиты.".

2. Смоделировать инвентарь и действия игрока:
   - Создайте список List<Item>, который будет играть роль инвентаря.
   - Положите в инвентарь (добавьте в список) по одному экземпляру каждого типа предметов 
     с произвольными названиями и характеристиками.
     - Пример: 
       Меч (Вес: 5, Атака: 15), 
       Малое зелье здоровья (Вес: 1, Лечение: 50), 
       Стальной щит (Вес: 8, Защита: 20).
   - Выведите на экран содержимое инвентаря (просто названия).

3. Применить upcasting (это уже произошло в списке):
   - Объясните (сами себе), почему мы смогли положить Weapon, Potion и Armor в список List<Item>. 
     Что произошло с типами?

4. Применить downcasting (главная задача):
   - Игрок хочет использовать каждый предмет в инвентаре.
   - Пройдитесь по инвентарю циклом foreach.
   - Для каждого предмета нужно:
     - Вызвать полиморфный метод Use(). 
       (Посмотри, что будет, если просто вызвать item.Use()).
     - Дополнительно: Если предмет является оружием (Weapon), 
       то после использования нужно вывести в консоль: 
       "Это оружие! Его сила атаки: X" (где X — значение поля AttackPower). 
       Это можно сделать только после приведения типа.
     - Безопасно: Реализуй это двумя способами (можно в разных циклах или через if):
       a) С использованием ключевого слова as.
       b) С использованием ключевого слова is с паттерном (как в современном C#).

Примерный ожидаемый вывод в консоль:

Содержимое инвентаря:
- Стальной меч
- Малое зелье здоровья
- Щит стража

Используем предметы:
Стальной меч наносит 15 урона.
Это оружие! Его сила атаки: 15
Малое зелье здоровья восстанавливает 50 здоровья.
Щит стража дает 20 защиты.

Общий вес инвентаря: 14 кг.


Подсказки:

- Для безопасного приведения используй as. 
  Если приведение не удалось, переменная станет null.

- Для проверки типа используй is. Например: 
  if (item is Weapon weapon) { ... } - это сразу создаст переменную weapon нужного типа.
