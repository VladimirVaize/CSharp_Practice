Система порядка хода в пошаговой RPG

Контекст:

Ты разрабатываешь пошаговую RPG. В бою порядок хода определяется инициативой персонажей. 
Чем выше инициатива, тем раньше персонаж ходит. Но в процессе боя инициатива может меняться:

- Персонаж применил ускоряющее заклинание → инициатива повысилась

- Персонаж получил замедляющий эффект → инициатива понизилась

- Персонаж был оглушен → временно теряет ход

Каждый раз, когда инициатива меняется, список очередности хода должен быть пересортирован. 
Но полная пересортировка сложными алгоритмами (QuickSort) — это избыточно, ведь изменения обычно небольшие. 
Гномья сортировка идеально подходит для этого сценария, так как она эффективна для почти отсортированных списков.

Что нужно сделать:
1. Создай класс Combatant (участник боя)
   У него должны быть:
   - string Name - имя персонажа
   - int Initiative - инициатива (определяет порядок хода, чем выше, тем раньше ходит)
   - bool IsStunned - оглушен ли (если true, то пропускает ход)
   - int Health - здоровье (для реализации эффектов)
   - Метод ApplyEffect(EffectType effect, int value) - применяет эффект, меняющий инициативу

2. Создай перечисление EffectType
   - Haste - ускорение (+инициатива)
   - Slow - замедление (-инициатива)
   - Stun - оглушение (IsStunned = true на N ходов)

3. Создай класс TurnOrder
   Это главный класс, который управляет порядком хода. В нем должно быть:
   - List<Combatant> combatants - список всех участников боя
   - Метод SortByInitiative() - использует гномью сортировку для упорядочивания списка по убыванию инициативы 
     (от большего к меньшему)
   - Метод UpdateInitiative(Combatant target, EffectType effect, int value) - изменяет инициативу цели, 
     применяет эффект, а затем вызывает сортировку для обновления порядка
   - Метод GetNextTurn() - возвращает следующего участника, который может ходить (не оглушенного)
   - Метод DisplayOrder() - выводит текущий порядок хода в консоль с красивым форматированием

4. Реализуй гномью сортировку
   Алгоритм должен быть написан вручную, без использования List.Sort() или LINQ для сортировки 
   (LINQ можно использовать для вывода/поиска, но не для самой сортировки).

   Как работает гномья сортировка:
   index = 0
   while index < list.Count:
      if index == 0 or list[index] <= list[index - 1]:
         index++
      else:
         swap(list[index], list[index - 1])
         index--

   Для сортировки по убыванию инициативы условие будет:
   if index == 0 or list[index].Initiative <= list[index - 1].Initiative

5. Создай симуляцию боя в Main
   1. Создай 5-7 участников с разной инициативой.
   2. Выведи начальный порядок хода (после сортировки).
   3. Сымитируй 10 ходов боя:
      - Покажи, кто сейчас ходит.
      - Случайным образом выбери эффект (ускорение, замедление, оглушение) 
        и примени к случайному участнику (или к текущему).
      - Обнови порядок хода.
      - Выведи новый порядок.
   4. Обработай исключения (например, попытка применить эффект к мертвому участнику).

Бонусные задания (по желанию):

1. Сохранение состояния боя: 
   Добавь возможность сохранить текущее состояние боя (порядок хода, здоровье, эффекты) 
   в JSON файл и загрузить его при старте программы. Используй сериализацию.

2. Таймер хода: 
   Добавь асинхронную задержку между ходами с помощью await Task.Delay(), чтобы бой шел не мгновенно, а с паузой.

3. События: 
   Создай событие OnInitiativeChanged, которое срабатывает при изменении инициативы. 
   Подпиши на него метод вывода в консоль.

4. Анимация сортировки: 
   Выводи в консоль процесс перестановок при гномьей сортировке, 
   чтобы было видно, как элементы "двигаются" к своим местам.



Требования к коду:

- Используй обработку ошибок (try-catch) для потенциальных проблем (null-ссылки, выход за границы списка).

- Реализуй интерфейс IEffectApplier с методом Apply(Combatant target) для разных типов эффектов 
  (не обязательно, но будет плюсом).

- Используй обобщения если создаешь вспомогательные классы.
